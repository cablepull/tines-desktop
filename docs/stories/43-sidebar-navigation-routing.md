# Sidebar Navigation Routing

## Scenario
As a user of the Desktop IDE, the sidebar navigation buttons for Cases, Actions, and Settings are non-functional. I need them to route to actual pages so I can browse global actions, manage tenant settings, and navigate the full IDE without workarounds.

## Acceptance Criteria
- All sidebar buttons (Dashboard, Actions, Settings) navigate to their respective pages.
- The active page is visually highlighted in the sidebar.
- Navigation is instant (client-side state, no page reload).
- The "Cases" button is replaced with functional nav items.
